Parsing save data against balance data:

- On page load, the Balance version of the map is parsed, and each type of map element is parsed based on the first letter (Key). The colon-delimited string is split into an array at the colon.
  - Rock: stores index 1 as Id and index 2 as Level
  - Exit: stores index 2 as Level and index 3 as SecondaryId
  - CheckPoint: stores index 2 as Level and index 3 as SecondaryId
  - MineShaft: stores index 1 as Id and index 2 as Level and index 3 as SecondaryId and index 4 as TertiaryId and index 5 as TertiaryLevel
  - SpawningCart: stores index 1 as Id and index 2 as Level
- If save data is loaded, each element is then parsed according to the values in the Save File Key table.
  - Anything that's different in the save data and balance data (even if it went from non-blank to blank), save data takes priority.
- Each map element object will also need to contain its checkpoint index (how many checkpoints appear before it), its (row, column) index, and its integer index.
- Once all the available information has been established, the map renderer displays what it knows. For now, keep the same text content, and the tooltip will contain all the attributes of the grid object.
